What it is

A one-way broadcast intercom for households with speakers in more than one room. Amazon and Google both ship this inside their own walled gardens; nothing equivalent existed for Sonos, which is the gap I went after.

I built all of it: the FastAPI backend on Fly, the Sonos OAuth and audio pipeline, the web UI, the Capacitor iOS shell, the home-screen widget, and the App Store submission.

The decision

I built live audio streaming, then deleted it.

Live streaming was the obvious architecture, and I chose it for a reason: two-way intercom. Then the spike measured Sonos's playback buffer at 1 to 2 seconds, against the 400 milliseconds natural conversation needs. It is not reducible, because Sonos engineered it for gapless music. Two-way was dead. Which meant the reason I'd picked live streaming no longer existed.

What actually forced the rewrite was multi-tenancy. My two test households were remote, so I needed self-serve sign-up, and live streaming requires a per-user always-on relay. Recorded clips through Sonos's audioClip API don't.

Then the measurement surprised me. Clips measured 1 second end to end. Streaming measured 5. The 5-second floor I'd been designing around applied to stream buffering, not to audioClip, which Sonos built for low-latency notifications. The cheaper, more scalable, remote-capable path also happened to be the fastest one. I had been carrying a number from the wrong context for weeks.

What it cost

There is no two-way intercom, and there never will be on this architecture. That's a real ceiling on the product, not a phase-two item — two-way is a different product on different hardware.

I also pre-registered the trial's decision rule before running it, and excluded my own household from the read. My family using something I built is not evidence a market exists. That rule is the reason I still don't claim demand is proven: the honest status is shipped and unvalidated.

Status & next

Apple approved version 1.1 on 2026-06-29. It is live in 175 territories at $0.99. An in-app ratings prompt ships after a user's third successful broadcast. Sonos, Amazon Echo and Google Cast all work end to end for any household. Ring talk-back works too, but stays gated to my own household — it leans on an unofficial path I'm not willing to ship to strangers.

The open lever is demand, not distribution: recruit external households and read the pre-registered 14-day retention rule. If it fails, this stays a thing my family uses every day, which I'd already decided counts as a success.
